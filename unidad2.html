1. Anatomía de un proyecto Android

AndroidManifest.xml

Declara componentes, permisos y la actividad de inicio. Es la ficha de identidad de la aplicación.

res/layout

Contiene archivos XML que describen cómo se ve cada pantalla.

java/

Contiene la lógica, eventos, validaciones y comunicación con servicios.

Gradle

Administra la compilación y las dependencias del proyecto.

Relación esencialXML define la interfaz; Java controla el comportamiento; el Manifest registra la aplicación y sus permisos.

2. AndroidManifest.xml

Para el proyecto PicoSat, el Manifest declara la actividad principal y, más adelante, el permiso de Internet para conectarse a Firebase o a un servicio Wi‑Fi.

AndroidManifest.xml
<manifest xmlns:android="http://schemas.android.com/apk/res/android">
    <!-- Se utilizará en la unidad final para consultar datos en red -->
    <uses-permission android:name="android.permission.INTERNET" />

    <application
        android:allowBackup="true"
        android:label="PicoSat Monitor"
        android:theme="@style/Theme.PicoSat">
        <activity
            android:name=".MainActivity"
            android:exported="true">
            <intent-filter>
                <action android:name="android.intent.action.MAIN" />
                <category android:name="android.intent.category.LAUNCHER" />
            </intent-filter>
        </activity>
    </application>
</manifest>

3. Diseño XML y ConstraintLayout

ConstraintLayout posiciona cada componente mediante restricciones. Evita ubicar elementos con coordenadas fijas y facilita que la app se adapte a distintas pantallas.

activity_main.xml
<androidx.constraintlayout.widget.ConstraintLayout
    xmlns:android="http://schemas.android.com/apk/res/android"
    xmlns:app="http://schemas.android.com/apk/res-auto"
    android:layout_width="match_parent"
    android:layout_height="match_parent"
    android:padding="24dp">

    <TextView
        android:id="@+id/txtTitulo"
        android:layout_width="wrap_content"
        android:layout_height="wrap_content"
        android:text="Monitor PicoSat"
        android:textSize="28sp"
        android:textStyle="bold"
        app:layout_constraintTop_toTopOf="parent"
        app:layout_constraintStart_toStartOf="parent"
        app:layout_constraintEnd_toEndOf="parent" />

</androidx.constraintlayout.widget.ConstraintLayout>

4. Lógica Java

MainActivity es una pantalla de la app. En onCreate() se carga el XML con setContentView().

MainActivity.java
package com.umng.picosat;

import android.os.Bundle;
import androidx.appcompat.app.AppCompatActivity;

public class MainActivity extends AppCompatActivity {
    @Override
    protected void onCreate(Bundle savedInstanceState) {
        super.onCreate(savedInstanceState);
        setContentView(R.layout.activity_main);
    }
}

5. Configurar el emulador

Abre Device Manager

En Android Studio selecciona Tools → Device Manager.

Crea un dispositivo virtual

Escoge un teléfono de referencia y una imagen de sistema compatible.

Asigna recursos

Usa una configuración moderada; la guía recomienda un equipo con al menos 8 GB de RAM.

Ejecuta la app

Selecciona el dispositivo virtual y pulsa Run.

AlternativaTambién puedes activar las opciones de desarrollador y la depuración USB en un teléfono Android para probar la aplicación físicamente.

6. Ejercicio: primera pantalla PicoSat

Crea un proyecto llamado PicoSatMonitor, usa Java, selecciona una actividad vacía y construye una pantalla con título, subtítulo y un panel para la futura telemetría.
